CONTEXTO DEL PROYECTO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Proyecto: "Criminalística Cataluña"
Descripción: Análisis integral de criminalidad y población reclusa en Cataluña usando datos oficiales, machine learning y un chatbot local.

Backend disponible:
- API REST en Python/Flask que expone los datos procesados del MySQL star schema
- Modelos ML ya entrenados: K-Means (clusters de zona), Prophet (predicción temporal), XGBoost (probabilidad resolución), Profiler (perfil de caso)
- Chatbot local Ollama en puerto 11434 (debe estar en la misma red WiFi que el móvil)

Crímenes analizados: agresiones sexuales, homicidios, robo, robo con violencia e intimidación

Variables demográficas: nacionalidad, sexo, grupo de edad

Ámbito geográfico: Cataluña (comarcas y municipios) con profundización en Barcelona (barrios)

Período temporal: 2017–2024

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ARQUITECTURA DE LA APP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Navegación: Tab bar inferior con 5 pestañas principales + Stack navigator para vistas de detalle.

Gestión de estado: Context API para filtros globales (año, tipo crimen, comarca, sexo, edad, nacionalidad). Todos los hooks de datos usan estos filtros.

Llamadas API: Todas las pantallas llaman a http://:5000/api/... con fetch. Modo demo offline con datos JSON locales incluidos en el bundle si la API no responde.

PANTALLAS Y COMPONENTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TAB 1 — RESUMEN (icono: estadísticas)
Pantalla principal tipo dashboard ejecutivo:
- Header con logo del proyecto y selector de año (pill buttons horizontales: 2017-2024 + "Todo")
- Grid 2x2 de KPI cards: total incidentes, tasa resolución, total reclusos, comarca más peligrosa
- Gráfico de líneas (Victory Native LineChart) con evolución de los 4 tipos de crimen, leyenda interactiva
- Mini mapa de Cataluña (React Native Maps) con puntos de calor por comarca, navegable
- Tarjeta "Alerta ML" con la predicción de Prophet para el mes siguiente por tipo de crimen

TAB 2 — CRÍMENES (icono: lupa)
Análisis detallado de criminalidad:
- Filtro de tipo de crimen en la parte superior (chips seleccionables: Agresión sexual / Homicidio / Robo / Robo con violencia)
- BarChart horizontal con top 10 municipios por tipo de crimen seleccionado
- PieChart con distribución por nacionalidad (top 5 + "Otros")
- StackedBarChart con distribución por sexo y grupo de edad
- Heatmap de incidentes por día de semana y franja horaria (implementado con una grid de celdas coloreadas en React Native, sin librería externa)
- Lista scrollable de últimos registros con: tipo, ubicación, fecha, estado (resuelto/no resuelto)

TAB 3 — MAPA (icono: ubicación)
Análisis geoespacial interactivo:
- Mapa full-screen (React Native Maps) centrado en Cataluña
- Toggle superior: "Cataluña comarcas" / "Barcelona barrios"
- Polígonos de comarcas/barrios con relleno de color proporcional a la densidad de crimen (coroplético manual con coordenadas GeoJSON)
- Al tocar una comarca/barrio: bottom sheet deslizable con estadísticas de esa zona (incidentes por tipo, demografía, comparativa con media catalana, cluster asignado por K-Means)
- Botón flotante para alternar entre vista de crímenes y vista de centros penitenciarios
- En modo centros penitenciarios: markers con icono personalizado, al tocar muestra capacidad, ocupación, distribución de internos

TAB 4 — RECLUSOS (icono: institución)
Análisis de población penitenciaria:
- KPI cards: total reclusos Cataluña, % extranjeros, % hombres, evolución vs año anterior
- HorizontalBarChart con reclusos por centro penitenciario, coloreado por % extranjeros
- SegmentedControl: ver por Sexo / Edad / Nacionalidad / Delito
- Según segmento seleccionado, cambia el gráfico inferior (PieChart o BarChart)
- Pantalla de detalle por centro (navegando con Stack): nombre, municipio, capacidad, ocupación actual, distribución demográfica completa, gráfico de evolución histórica
- Comparativa "crimen vs reclusión": para el tipo de crimen seleccionado, muestra side-by-side el perfil demográfico de quien comete el crimen vs. quien está preso por ese crimen

TAB 5 — IA (icono: chatbot)
Interfaz dual de inteligencia artificial:

MODO CHAT (por defecto):
- Interfaz de chat completa estilo messenger
- FlatList con mensajes de usuario (derecha, fondo azul) y asistente (izquierda, fondo gris oscuro)
- TextInput en la parte inferior con botón enviar y botón micrófono (placeholder, sin implementar)
- Antes de enviar a Ollama, fetch a la API Flask con palabras clave del mensaje para obtener datos relevantes e inyectarlos en el system prompt
- Indicador de escritura animado (tres puntos pulsantes) mientras espera respuesta
- Botones de acceso rápido encima del input: "¿Dónde más homicidios?", "Tendencia agresiones", "Perfil reclusos", "Predicción próximo mes"
- Historial de conversación persistente en AsyncStorage durante la sesión

MODO PERFILADOR (toggle en el header):
- Formulario paso a paso (3 pasos con progress bar):
  Paso 1: tipo de crimen + comarca/municipio
  Paso 2: franja horaria + día de semana
  Paso 3: (opcional) datos demográficos del sospechoso si los hay
- Botón "Analizar caso" que llama al endpoint /api/profiler
- Pantalla de resultado con: cluster asignado, características del cluster, probabilidad de resolución (gauge animado), casos históricos similares (lista), recomendaciones estadísticas

REQUISITOS TÉCNICOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Framework: React Native con Expo SDK 51+
Navegación: @react-navigation/native + @react-navigation/bottom-tabs + @react-navigation/stack
Gráficos: victory-native (BarChart, LineChart, PieChart, VictoryArea)
Mapas: react-native-maps con Google Maps (Android) y Apple Maps (iOS)
Bottom sheet: @gorhom/bottom-sheet
Almacenamiento local: @react-native-async-storage/async-storage (historial chat)
HTTP: fetch nativo (sin axios)
Estado global: React Context API + useReducer para filtros

Estructura de archivos:
/src
  /screens (una carpeta por tab)
  /components (componentes reutilizables)
  /context (FilterContext, ChatContext)
  /hooks (useCrimes, useReclusos, useProfiler, useChat)
  /services (api.js con todas las llamadas fetch)
  /data (JSONs de demo offline)
  /constants (colors.js, typography.js, endpoints.js)
  /navigation (AppNavigator.js)

En endpoints.js define:
const BASE_URL = 'http://192.168.1.X:5000' // cambiar por IP local real
const OLLAMA_URL = 'http://192.168.1.X:11434'
